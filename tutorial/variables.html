C# Variables

Variabelen zijn containers ('doosjes') om data in op te slaan.

Een variabele maken

In C# moet je altijd vertellen wat er in de doos zit (het Type) en hoe hij heet (de Naam).

string naam = "Ali";
int leeftijd = 25;
bool isStudent = true;

Het keyword var

Als de compiler zelf kan raden wat het type is, mag je var gebruiken.

var stad = "Amsterdam"; // Compiler weet: dit is een string
var score = 100;        // Compiler weet: dit is een int
â„¹ Best Practice:
Gebruik var alleen als het overduidelijk is wat het type is.

Constants

Als een waarde nooit mag veranderen, gebruik je const.

const double PI = 3.14159;
PI = 4; // FOUT! Je mag dit niet aanpassen.

Naamgeving (Naming Conventions)

In C# zijn we streng over hoe je dingen noemt:

  • camelCase (voor lokale variabelen): mijnNaam, aantalKlanten
  • PascalCase (voor Classes en Methods): BerekenTotaal(), KlantService
  • Gebruik Engels in professionele code (wij gebruiken hier NL voor de uitleg).

Deep Dive: `var` vs Explicit Typing

Wanneer gebruik je `var`? De regel is simpel: Gebruik het alleen als de rechterkant van de vergelijking overduidelijk maakt wat het type is.

var name = "Ali"; // Obvious String
var age = 25;     // Obvious Int
var list = new List<string>();

🎯 Senior Pro Tip: Naming

In professionele teams word je ontslagen als je variabelen `x`, `y` of `data` noemt. Variabelen moeten beschrijvend zijn.

Fout: `int d = 7;`
Goed: `int daysInWeek = 7;`

🎯 Extra Challenge: The Swap

Stel je hebt `int a = 10;` en `int b = 20;`. Schrijf code om de waardes om te wisselen zodat `a` 20 wordt en `b` 10. Je mag de getallen niet direct toewijzen!

Toon Antwoord 
int a = 10;
int b = 20;

// Gebruik een tijdelijke variabele (temp)
int temp = a;
a = b;
b = temp;

Console.WriteLine(a); // 20
Console.WriteLine(b); // 10
