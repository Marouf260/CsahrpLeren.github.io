1.3 De Garbage Collector (GC)

In C++ moet je zelf geheugen opruimen (`delete`). In C# doet de GC dat. Maar... niet gratis.

Hoe de GC werkt (Simpel)

De GC kijkt naar alle objecten op de Heap. Hij vraagt: "Is dit object nog bereikbaar vanuit de Stack (root)?"

  • Ja? Markeer als 'levend'.
  • Nee? Ruim op en compacteer het geheugen.

Wanneer de GC draait, kan hij je hele applicatie pauzeren ("Stop the World"). Dit duurt vaak maar milliseconden, maar bij games of high-frequency trading is dit dodelijk.

Generaties (Gen 0, 1, 2)

.NET optimaliseert met de aanname: "Die jongen sterven jong". De meeste objecten zijn tijdelijk.

  1. Gen 0 (Nieuw): Hier komen nieuwe objecten (klein). GC runt hier heel vaak en heel snel.
  2. Gen 1 (Overlevers): Objecten die Gen 0 overleven verhuizen hierheen.
  3. Gen 2 (Oud): Objecten die Gen 1 overleven. Hier zitten statische variabelen, caches, etc. GC runt hier zelden, want het is duur (full sweep).
üõë Code Smell: Allocations in Loops
// SLECHT:
void UpdateLoop() { // 60 frames per seconde
    var list = new List(); // Allocatie in Gen 0 elke frame!
    // ...
}

// GOED:
List reusableList = new List(); // Gen 2 (blijft leven)
void UpdateLoop() {
    reusableList.Clear(); // Geen allocatie, alleen hergebruik
    // ...
}

De slechte code dwingt de GC om Gen 0 constant te cleanen. De goede code alloceert 0 bytes per frame.
