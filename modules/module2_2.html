2.2 Class, Struct of Record?

C# geeft je drie smaken om data te modelleren. Wanneer kies je welke?

1. Class (Reference Type)

De standaard. 90% van je code zal classes zijn. Ze leven op de Heap en ondersteunen overerving (Inheritance). 

public class Persoon {
    public string Naam { get; set; }
}

2. Struct (Value Type)

Voor kleine, atomische data die zich gedraagt als een getal. Leeft op de Stack (als local var) of inline in een array.

public struct Punt {
    public int X;
    public int Y;
}

3. Record (Reference Type*)

Nieuw in C# 9. Records zijn classes, maar met Value Semantics voor gelijkheid.

public record Klant(string Naam, int Leeftijd);

var k1 = new Klant("Ali", 25);
var k2 = new Klant("Ali", 25);

Console.WriteLine(k1 == k2); // TRUE! (Bij classes zou dit False zijn)

Records zijn perfect voor DTO's (Data Transfer Objects): simpele pakketjes data zonder gedrag.

*Er bestaat ook `record struct`, maar dat is advanced.

Beslisboom

Vraag Kies...
Heeft het een identiteit? (zoals een User in database) Class
Is het alleen data om door te sturen? (API response) Record
Is het kleiner dan 16 bytes, onveranderlijk en 'wiskundig'? Struct
