4.1 De Call Stack Visualized

Kijk eens naar de debugger in Visual Studio. Dat venster "Call Stack" is geen magie, het is een weergave van het RAM.

Hoe het werkt

Stel we hebben deze code:

void Main() {
    A();
}
void A() {
    int x = 10;
    B(x);
}
void B(int val) {
    // Breakpoint hier
}

De Stack op moment van Breakpoint:

STACK (Groeit naar beneden)
[ Frame: Main ]
[ Frame: A ]
x = 10
Return Address: Main rules 2
[ Frame: B ] (Active)
val = 10
Return Address: A regel 6

Recursie & Stack Overflow

Als method A zichzelf aanroept, komt er telkens een nieuw frame bovenop. Omdat de Stack beperkt is (1MB), is hij na een paar duizend aanroepen vol.

void Boom() {
    Boom(); // PUSH, PUSH, PUSH...
}
// Resultaat: StackOverflowException. App crasht direct.
