0.2 Stack vs Heap: De Twee Werelden

Als programmeur ben je eigenlijk een geheugen-manager. Je moet precies weten waar je data leeft.

‚ùå üî• Cruciaal Concept:
Het verschil niet snappen tussen Stack en Heap is de #1 reden voor bugs en trage applicaties in C#.

De Stack (De Werkbank)

De Stack is een blok geheugen dat gereserveerd is voor draden van uitvoering (threads). Het werkt extreem simpel: Last In, First Out (LIFO).

  • Snelheid: Extreem hoog. Allocatie is simpelweg een pointer verplaatsen.
  • Levensduur: Automatisch opgeruimd zodra de method klaar is (popped).
  • Grootte: Klein (meestal 1MB per thread).
  • Wat komt hier? Value Types (int, bool, structs) en pointers naar de Heap.
void Main() {
    int x = 10;      // Stack
    bool y = true;   // Stack
    Calculate(x);    // Nieuw Stack Frame bovenop Main
} // x en y bestaan hier niet meer!

De Heap (Het Magazijn)

De Heap is een grote, ongestructureerde berg geheugen voor data die langer moet leven dan √©√©n methode-aanroep.

  • Snelheid: Trager. De Runtime moet zoeken naar een vrije plek.
  • Levensduur: Blijft bestaan tot de Garbage Collector (GC) het opruimt.
  • Grootte: Groot (beperkt door fysiek RAM/OS).
  • Wat komt hier? Reference Types (Classes, Strings, Arrays).

De variabele u op de Stack bevat enkel het adres van het object op de Heap.

Stack Overflow?

Als je oneindige recursie hebt (method roept zichzelf aan zonder stop), raakt de Stack vol (1MB limiet). Resultaat: StackOverflowException. De Heap raakt minder snel vol (Out Of Memory).

Toon Antwoord

Op regel 1 leeft a op de Stack.
Op regel 3 wordt a gekopieerd naar de list, die op de Heap leeft. (Dit heet technisch gezien geen 'boxing' omdat List<int> generiek is, maar de waarde staat nu wel in Heap-geheugen).
