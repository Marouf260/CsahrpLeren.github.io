2.3 The Billion Dollar Mistake (Null)

Tony Hoare verontschuldigde zich voor het uitvinden van null. In C# 8.0 hebben we het eindelijk "opgelost".

Het probleem

string s = GetString();
// Crash: Object reference not set to an instance of an object.
int lengte = s.Length;

Vroeger kon elk reference type null zijn. Je moest overal checks inbouwen.

De Oplossing: Nullable Reference Types

In moderne .NET (sinds C# 8) is dit de regel:

‚úÖ Regel:
Reference types zijn standaard NIET null. 
string s = null;  // Compiler Warning! Mag niet!
string? s = null; // OK, je zegt expliciet dat het null mag zijn.

Operators die je moet kennen

Als je met nullable types werkt, heb je gadgets nodig:

1. Null-Conditional Operator (`?.`)

string? s = null;
int? lengte = s?.Length; // Als s null is, wordt lengte null. Geen crash.

2. Null-Coalescing Operator (`??`)

string? s = null;
// Als links null is, pak rechts
string safe = s ?? "Fallback Waarde";

3. Null-Forgiving Operator (`!`)

string? s = "Ik weet zeker dat dit niet null is";
// "Bek houwe compiler, ik weet wat ik doe"
int lengte = s!.Length;
‚ùå Pas op:
Gebruik `!` alleen als je 1000% zeker bent. Als je het mis hebt, crash je alsnog.
