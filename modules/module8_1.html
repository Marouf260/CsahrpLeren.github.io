8.1 Deferred Execution

LINQ queries zijn lui. Heel lui. En dat is goed.

Het Concept

var numbers = new List { 1, 2, 3 };

// 1. De query wordt hier GEBOUWD, niet uitgevoerd!
var query = numbers.Where(n => n > 1);

numbers.Add(4);

// 2. Hier wordt de query pas uitgevoerd (ge√Øtereerd).
// Verrassing: 4 zit ook in het resultaat!
foreach (var n in query) {
    Console.WriteLine(n); // Print: 2, 3, 4
}

Hoe werkt dit? (Yield Return)

Onder water gebruikt LINQ yield return. Dit bouwt een State Machine. De code loopt stap voor stap, alleen wanneer de consument (de foreach loop) om het volgende item vraagt.
