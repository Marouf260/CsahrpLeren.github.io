6.1 Interfaces zijn Contracten

Een interface definieert WAT iets kan, niet HOE.

Dependency Inversion (De 'D' van SOLID)

Hard-coded dependencies zijn dodelijk voor onderhoud.

// SLECHT: Directe afhankelijkheid
class Winkel {
    private CashRegister _kassa = new CashRegister(); // Zit vast aan 1 merk kassa
}

// GOED: Interface
interface IPaymentTerminal {
    void Pay(decimal amount);
}

class Winkel {
    private IPaymentTerminal _terminal;
    
    // Dependency Injection (Constructor Injection)
    public Winkel(IPaymentTerminal terminal) {
        _terminal = terminal;
    }
}

Nu kan de Winkel werken met een CashRegister, een PinApparaat, of een FakeTerminal (voor tests).
