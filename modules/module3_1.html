3.1 Branching & Pattern Matching

De kunst van beslissingen nemen zonder spaghetti te maken.

1. Guard Clauses (Clean Code)

Voorkom diepe nesting ("Arrow Code"). Draai je if om en return vroeg.

// ‚ùå BAD: Arrow Code
void Process(User user) {
    if (user != null) {
        if (user.IsActive) {
            if (user.HasAccess) {
                // Do work...
            }
        }
    }
}

// ‚úÖ GOOD: Guard Clauses
void Process(User user) {
    if (user == null) return;
    if (!user.IsActive) return;
    if (!user.HasAccess) return;

    // Do work... (Lekker vlak)
}

2. Pattern Matching (Modern C#)

Vergeet complexe if/else ketens. Gebruik patronen.

object data = "Hello";

// C# 9+
if (data is string s && s.Length > 5) {
    Console.WriteLine($"Lang woord: {s}");
}

// Switch Expression (Super strak)
string status = score switch {
    < 0 => "Error",
    0 => "Start",
    < 50 => "Normaal",
    _ => "Hoog"
};

3. Deep Dive: Branch Prediction

‚Ñπ üß† CPU Weetje:
De CPU probeert te gokken welke kant een if op gaat VOORDAT hij het antwoord weet.

Als hij goed gokt: 0 vertraging. Als hij fout gokt: De hele pijplijn moet leeggemaakt worden (duur!).

Daarom is het verwerken van een gesorteerde array vaak sneller dan een ongesorteerde array, puur omdat de if (x > 50) voorspelbaarder wordt (eerst allemaal false, dan allemaal true).
