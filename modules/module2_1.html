Het Kopieer-Gedrag (Demo)

Dit is waar bugs ontstaan. Kijk goed naar het verschil:

// --- VALUE TYPE (int) ---
int x = 10;
int y = x;  // Kopieert de waarde 10
y = 20;     // y is nu 20, x blijft 10!
// Resultaat: x=10, y=20 (Veilig, onafhankelijk)

// --- REFERENCE TYPE (List) ---
List a = new List { 1 };
List b = a; // Kopieert de POINTER (het adres 0x123)
b.Add(2);        // We voegen toe via b...
// Resultaat: a heeft nu OOK 2 items! Want a en b wijzen naar hetzelfde huis.
âŒ Valkuil:
Als je een object doorgeeft aan een methode, geeft je de afstandsbediening door, niet het object. De methode kan jouw object dus aanpassen!

Wanneer kies je wat?

Microsoft heeft richtlijnen ("Guidelines"):

  • Gebruik Class (Ref Type) voor bijna alles: Entiteiten, Business Logic, Services.
  • Gebruik Struct (Value Type) alleen als:
    • Het object klein is (< 16 bytes).
    • Het onveranderlijk (immutable) is.
    • Het logisch gezien een "waarde" is (zoals Point, Color, ComplexNumber).
