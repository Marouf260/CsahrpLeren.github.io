1.4 Anatomie van een .NET Project

Wat zit er eigenlijk in die folders?

Het .csproj bestand

De project file is de blauwdruk. Het vertelt de SDK wat er moet gebeuren.

<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net8.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
  </PropertyGroup>
</Project>
  • OutputType: Exe (applicatie) of Library (dll).
  • TargetFramework: Welke versie van de Runtime heb je nodig?

Program.cs: Top-Level Statements

Sinds C# 9 ziet je Program.cs er misschien zo uit:

Console.WriteLine("Hello World");

Waar is de class? Waar is void Main? De compiler "vermomt" dit voor je. Achter de schermen maakt hij dit:

namespace MijnApp {
    internal class Program {
        static void Main(string[] args) {
             Console.WriteLine("Hello World");
        }
    }
}

Top-level statements zijn syntactic sugar om starters en script-schrijvers te helpen. Als engineer moet je weten dat de Main method er stiekem nog steeds is.

De 'bin' en 'obj' mappen

  • obj: Tijdelijke bestanden tijdens het compileren. Mag je verwijderen.
  • bin: Het eindresultaat. Hierin staat je .dll of .exe.
